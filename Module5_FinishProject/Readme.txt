
Перечень объектов.

1. Справочная информация (Reference Information)
    1.1 Класс Address               - класс для описания почтового адреса
    1.2 Класс ContactInformation    - класс для описания контактной информации. 
                                      Включает в себя свойство типа Address для заполнения
                                      почтового адреса
    1.3 Перечисление Positions      - перечень должностей организации
    1.4 Перечисление OrderStatuses  - перечень возможных статусов заказа

2. Контрагенты (Contractors)
    2.1 Класс Entity                - класс для описания организаций

3. Люди
    3.1 Класс Person                - абстрактный класс для описания человека
    3.2 Класс Employee              - класс для описания сотрудника магазина
    3.3 Класс Client                - класс для описания покупателя типа "Физическое лицо"

4. Типы доставки (Delivery)
    4.1 Класс Delivery              - абстрактный обобщенный класс для описания типов 
                                      доставки
    4.2 Класс HomeDelivery          - обобщенный класс для описания доставки до
                                      конкретного адреса. Доступен для организации и
                                      физического лица
    4.3 Класс PickPointDelivery     - класс для описания доставки до пункта выдачи.
                                      Доступен для физического лица
    4.4 Класс ShopDelivery          - класс для описания доставки до магазина.
                                      Доступен для физического лица

5. Заказ (Order)
    5.1 Позиции заказа (OrderItems)
        5.1.1 Класс OrderItem       - Абстрактный класс для описания позиции заказе
                                      (товар либо услуга)
        5.1.2 Класс Product         - класс для описания позиции заказа типа "Товар"
        5.1.3 Класс Service         - класс для описания позиции заказа типа "Услуга"
    5.2 Класс Order                 - абстрактный обобщенный класс для описания заказа
                                      покупателя. Может быть сформирован в зависимости
                                      от типа покупателя (организация/физическое лицо)
    5.3 Класс CustomerOrder         - класс для описания заказа покупателя.
                                      Формируется для клиента-физического лица.
